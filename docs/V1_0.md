## Definition of success

REX succeeds when an operator can **dogfood** from the IDE on macOS: a **supervised sidecar agent**, **brokered HTTP** inference, **daemon-enforced** stream and access policy, and **measurable economics** via the **observability suite**—Rex-owned storage, loopback read API, and bundled Grafana—without embedding vendor SDKs in clients or requiring operator-managed TSDBs. That matches [PURPOSE_AND_PRINCIPLES.md](PURPOSE_AND_PRINCIPLES.md).

**v1.0 Met** = every **Must** `RC-*` row below is **Met** (streaming/agent **RC-01–RC-10** plus observability **RC-S3–RC-S5**)—then create the **`v1.0.0` git tag**. Not a production SLA or supported distribution.

**v1.0 status today:** **Not Met** — observability Must rows **RC-S3** (**Partial**), **RC-S4** (**Not met**), **RC-S5** (**Not met**) remain open. Workspace **`1.0.0`** in [Cargo.toml](../Cargo.toml) is preparatory; the **git tag is blocked** until those rows close.

## Public API at 1.0.0 (Semantic Versioning)

| Version | Meaning for REX |
|---------|-----------------|
| **`0.y.z`** | Initial development; public API **may change** without a major bump. |
| **`1.0.0` (workspace version in [Cargo.toml](../Cargo.toml) today)** | Public API **stable**; breaking client-visible changes require a **major** bump and documented migration. The **`v1.0.0` git tag** is created when a maintainer merges the release-plz Release PR **after** all Must **RC-*** rows are **Met** (no `v*` tag on `main` until then). |

**Public API boundary** (see [DEVELOPER_EXPERIENCE_GUIDE.md](DEVELOPER_EXPERIENCE_GUIDE.md) §5):

- `proto/rex/v1/rex.proto` and generated **`rex.v1`** surfaces consumed by the unified **`rex`** CLI and the extension.
- **`rex complete --format ndjson`** (shim: `rex-cli complete`): one terminal `done` or `error` per stream; conformance in [fixtures/ndjson_contract/](../fixtures/ndjson_contract/) and [EXTENSION.md](EXTENSION.md).
- **`rex.sidecar.v1`**: control plane distinct from `rex.v1`. Additive verbs and fields are allowed in minor releases; breaking changes require a **versioned** sidecar API migration (same discipline as `rex.v1`).

**Not implied by `1.0.0`:** long-term support, zero defects, **RC-S6** live LLM smoke closure, remote TLS/auth, or withdrawal of the study-project framing in the purpose doc.

## What v1.0 does not promise

- Production SLA or commercial support.
- Apple MLX, multi-plugin fleets, durable long-term memory, MCP implementation, on-disk `rex config`, or Node gRPC streaming in the extension.
- VM/container as the default Mac sidecar envelope.
- Required **OpenTelemetry Collector**, **Prometheus**, **Loki**, or **Tempo** for the product UI path—Rex owns storage and the read API ([ADR 0026](architecture/decisions/0026-rex-owned-storage-grafana-otel-datasource.md)).
- **CHCE mmap** as the JSON default store engine before **R054** promotion gates pass.
- Live LLM on every PR (**RC-10** preserved); opt-in smoke is **RC-S6** / **R039–R040**.

Deferred items: [ROADMAP.md](ROADMAP.md) **Later** and **Parked**.

## When to tag `1.0.0`

1. Confirm every Must **RC-*** row is **Met** in the tables above (**RC-01–RC-10** and **RC-S3–RC-S5**).
2. Merge the **release-plz** Release PR that bumps workspace `version` to **`1.0.0`** in [Cargo.toml](../Cargo.toml) (do not merge that PR before step 1). See [RELEASE.md](RELEASE.md).
3. Note extension release policy in [EXTENSION_RELEASE.md](EXTENSION_RELEASE.md) (extension may keep independent `rex-vscode-v*` tags; reference workspace v1.0 in release notes when aligned).
